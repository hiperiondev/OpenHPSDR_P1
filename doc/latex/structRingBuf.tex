\doxysubsection{Detailed Description}
Ring buffer element type

The type of the ring buffer elements is not critical for the lock-\/free operation and does not need to be \char`\"{}atomic\char`\"{}. For example, it can be an integer type (uint16\+\_\+t, uint32\+\_\+t, uint64\+\_\+t), a pointer type, or even a struct type. However, the bigger the type the more RAM is needed for the ring buffer and more CPU cycles are needed to copy it into and out of the buffer memory.
